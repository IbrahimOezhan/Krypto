Linux Bash and Cryptography Essentials

This page combines Bash scripting concepts with foundational cryptography knowledge.

1. Cryptography Basics

Cryptography enables secure communication over insecure channels and addresses key objectives:

  • Confidentiality: Ensures that data is accessible only to authorized users.
  • Integrity: Prevents undetected modification of data.
  • Non-repudiation: Validates the authenticity of a message, ensuring the sender cannot deny authorship.

1.1 Encryption and Decryption

Encryption transforms plaintext into unreadable ciphertext using a key, while decryption reverts it back to plaintext.

# Example using OpenSSL
# Encrypt a message
openssl enc -aes-256-cbc -salt -in message.txt -out message.enc

# Decrypt the message
openssl enc -aes-256-cbc -d -in message.enc -out message.txt

1.2 Types of Cryptographic Algorithms

  • Symmetric Cryptography: Uses the same key for encryption and decryption (e.g., AES).
  • Asymmetric Cryptography: Uses a public/private key pair for encryption and decryption (e.g., RSA).

1.3 Kerckhoff's Principle

The security of a cryptographic system should rely on the secrecy of the key, not the algorithm. Public algorithms are preferred over obscurity for transparency and trust.

1.4 Cryptographic Hash Functions

Hash functions generate fixed-length digests from input data. They are deterministic, collision-resistant, and irreversible.

# Generate a SHA-256 hash
sha256sum <<< "Example text"

2. Public Key Infrastructure (PKI)

PKI establishes trust through certificate authorities (CAs) that verify identities and sign digital certificates.

  • Digital Certificates: Bind public keys to entities and are signed by CAs.
  • Certificate Validation: Involves checking the certificate chain and verifying the CA's signature.

3. Key Differences Between Bash Operators

Input Redirection

  • <: Redirects file content as input to a command.
  • <<: Here-document provides inline input.
  • <<<: Here-string provides a single-line input.

Output Redirection

  • >: Overwrites output to a file.
  • >>: Appends output to a file.

Pipes and Process Substitution

  • |: Sends the output of one command to another.
  • |&: Redirects both stdout and stderr to another command.

4. Combining Bash and Cryptography

Bash scripting simplifies cryptographic tasks through automation. For example:

# Automated encryption and decryption
for file in *.txt; do
    openssl enc -aes-256-cbc -salt -in "$file" -out "$file.enc"
    echo "$file encrypted."
done
